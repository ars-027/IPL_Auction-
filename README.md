# IPL_Auction-
BIDBLAZE OS: Enterprise Auction War Room
**BIDBLAZE OS** is a real-time, quantitative auction simulation and franchise management dashboard. Designed like a professional sports War Room, it combines a real-time Node.js/Socket.io backend with a Python-powered Machine Learning data pipeline to deliver dynamic player valuations, enforce strict roster constraints, and track global market intelligence.

## 🌟 Key Features

* **🧠 Live Neural Valuation Engine:** Uses "Moneyball" logic to calculate a player's intrinsic market value. It dynamically scans rival franchises' remaining purses, evaluates positional scarcity (V.O.R.P.), and warns you of capital exhaustion before you bid.
* **🔄 RTM Omni-Tracker:** Visually tracks "Right to Match" (RTM) cards across all franchises. Badges update live to show if an RTM player is *Available*, *Acquired*, or *Stolen* by a rival.
* **📊 Python Data Pipeline:** A bulletproof Python engine parses raw `.csv` files. It features a **Dynamic Imputation Engine** that automatically calculates missing ("TB REVEALED") player points based on their Base Price, Role, and PM Tier multiplier.
* **🛡️ Strict Top 7 Mission Control:** Mathematically enforces official roster rules (3 Batsmen, 2 Bowlers, 1 Wicketkeeper, 1 All-Rounder) and caps Overseas players at 3 within the core squad.
* **📈 Global Market Intelligence:** Live leaderboards track the Purchasing Power Parity (PPP) of every franchise, flagging teams that are hyper-inflated or dangerously illiquid.

## 🏗️ Architecture & Tech Stack

* **Frontend:** Vanilla JavaScript, HTML5, Tailwind CSS, Socket.io-client
* **Backend:** Node.js, Express, Socket.io (Handles state management and live WebSockets)
* **Data Engine:** Python, Pandas, Regex (Handles raw CSV parsing, data cleansing, and statistical imputation)

🚀 Installation & Setup
Prerequisites
Node.js (v14 or higher)

Python (v3.8 or higher)

Python pandas library (pip install pandas)

Step-by-Step Guide
1. Clone the repository:

Bash
git clone [https://github.com/yourusername/bidblaze-os.git](https://github.com/yourusername/bidblaze-os.git)
cd bidblaze-os
2. Install Node dependencies:

Bash
cd backend
npm install express socket.io cors
3. Install Python dependencies:

Bash
pip install pandas
4. Add your Data:
Ensure your player files (BAT.csv, BOWL.csv, WK.csv, AR.csv) are placed inside the data/ folder. The Python script expects headers like Player, Role, Points, Origin, Base Price, RTM Franchise, and PM Tier.

5. Start the Server:

Bash
cd backend
node server.js
6. Launch the Dashboard:
Open frontend/index.html in your favorite web browser (or serve it via a tool like Live Server).

Note: Upon loading the UI, select your franchise, set your purse constraints, and click BOOT SEQUENCE. This triggers the Node server to execute the Python script, compiling your .csv files into a live, normalized JSON database.

⚙️ How the Valuation Engine Works
The dashboard calculates Maximum Recommended Bids using a 5-pillar logic system:

Intrinsic Point Curve: Exponentially scales player points (e.g., a 98-point player is worth vastly more than a 90-point player).

Positional Scarcity: Checks the gap between the target player and the next best available backup in the database.

Rival Threat Assessment: Scans the purses of only the rival franchises who mathematically still need that specific role.

Bench Penalties: If your core Top 7 is already full, the AI sharply downgrades your max bid, warning you that you are buying for the bench.

Absolute Ceiling Check: Ensures that no suggested bid will leave you mathematically incapable of buying the rest of your required 14-man roster at base price (₹0.5 Cr).

🐛 Troubleshooting
Server crashes on boot: Ensure you have Python installed and added to your system PATH. The Node server uses child_process.exec to call python.

Players missing / UI stuck on 75 points: Click the red ABORT SESSION button in the top right of the UI. This deletes the cached global_auction_state.json file and forces the Python engine to re-parse the raw CSVs from scratch.
